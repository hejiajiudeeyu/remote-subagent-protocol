# Support

## 1. 使用建议

根据问题类型选择渠道：
- 使用问题 / 接入咨询：GitHub Discussions（建议）
- 缺陷报告：GitHub Issues
- 安全漏洞：见 `SECURITY.md`，请通过 GitHub PVR 私下报告
- 社区交流（非敏感）：Discord `https://discord.gg/e4nDukFAc6`

## 2. 提问前检查

请先确认：
- 已阅读 `README.md` 与 `docs/` 对应章节
- 已搜索现有 Issues / Discussions
- 能提供最小复现信息（输入、期望、实际）

## 3. 最小问题模板

建议包含：
- 背景与目标
- 当前行为
- 期望行为
- 复现步骤
- 相关日志或截图

## 4. 响应说明

这是一个早期项目，维护者会尽量及时响应，但不保证实时支持。

## 5. 渠道边界

- 架构决策、需求变更、代码评审：必须回到 GitHub（Issue/PR）。
- Discord 用于实时讨论、答疑和协作组织，不作为最终决策记录。
- 安全问题不要在 Discord 或公开 Issue 披露细节。
