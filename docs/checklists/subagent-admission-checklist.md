# Subagent 上架准入清单（v0.1）

状态：Draft  
更新时间：2026-03-03  
适用对象：平台审核、卖家自检、灰度准入评估

## 1. 基础信息完整性（必填）

- [ ] `subagent_id`、`seller_id`、`display_name`、`description` 完整
- [ ] `capabilities`、`supported_task_types` 可读且可检索
- [ ] `template_ref` 可访问，模板 5 件套完整
- [ ] `delivery_address`、`contact` 可用

## 2. 五问门槛（全部为是）

- [ ] 可合约化：输入/输出/约束已 schema 化
- [ ] 可异步化：业务可接受异步时延
- [ ] 可幂等化：同一 `request_id` 可安全重放
- [ ] 可验收化：有明确验收规则（签名/结构/质量阈值）
- [ ] 可控风险：预算、超时、失败回退、人工兜底已定义

若任一项为否：不通过常规上架，进入修订或白名单流程。

## 3. 风险与合规（必过）

- [ ] 无不可逆高风险动作（或已隔离为人工确认步骤）
- [ ] 数据处理边界明确（输入数据范围、保留策略、脱敏策略）
- [ ] token claims 校验链路清晰（`aud/sub/request_id/subagent_id/exp`）
- [ ] 结果签名机制可验，公钥可轮换

## 4. 可运维性（必过）

- [ ] 超时与重试策略明确（含最大重试次数）
- [ ] Buyer 超时语义明确（`soft_timeout` 询问、`hard_timeout` 本地终态，不承诺远端 kill）
- [ ] 指标上报可用（成功率、超时率、执行时延）
- [ ] 故障可定位（最小日志与错误码规范）
- [ ] `availability_status` 更新机制已接入（心跳或等价机制）

## 5. 分类归档（至少 1 类）

- [ ] 文档与格式工程
- [ ] 内容渠道化生产
- [ ] 专业领域排障与诊断
- [ ] 基础设施密集型任务
- [ ] 标准化批处理任务
- [ ] 其他（需写明理由）

## 6. 准入结论

- [ ] 通过：可全量上架
- [ ] 有条件通过：灰度上架（限定买家/任务类型/预算）
- [ ] 不通过：返回修订

审核备注建议包含：
- 不通过或灰度的具体原因
- 需要补齐的字段与验证项
- 复审触发条件与目标时间
